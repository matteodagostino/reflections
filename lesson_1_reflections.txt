What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

    Il salvataggio automatico sembra migliore ma salverà molte versioni inutili
    e nel caso della programmazione mi interessa avere punti versioni legate
    a modifiche di una certa rilevanza logica per non perdere il controllo
    della storia e del versioning

Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

    I sistemi di controllo versione legati alla programmazione necessitano
    di tracciare modifiche logiche ottenute modificando più file, mentre 
    i sistemi legati a documenti specifici non necessitano di questo

How can you use the commands git log and git diff to view the history of files?

Git log mi permette di vedere la lista di tutti i commit. Tramite l'ID di ogni 
commit e tramite il comando git diff potrò poi trovare le differenze specifiche
tra due commit
