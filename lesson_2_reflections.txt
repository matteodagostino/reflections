What happens when you initialize a repository? Why do you need to do it?

Quando viene inizializzata una repository viene cretao un file .git
contenente i meta dati necessari a tracciare la storia della repository, ma 
non viene creato nessun commit e vengono mostrati con git log i file
come ancora non tracciati (untracked files)

How is the staging area different from the working directory and the repository? What value do you think it offers?
La staging area è un'area di raccolta di file che precede l'impacchettamento in un commit. 
Nella working directory possono esserci molti file, ma inseriremo nella staging area solo i
file che intendiamo inviare per il prossimo commit.

How can you use the staging area to make sure you have one commit per logical change?
Con l'area di staging è possibile avere in un unico luogo tutti i file con le modifiche
effettuate, in modo da poter verificare le modifiche prima di fare un commit 

What are some situations when branches would be helpful in keeping your history organized? How would branches help?
I branches sono utili quando vogliamo testare due versioni differenti dello
stesso software senza per questo perdere la versione master o uno dei due test

How do the diagrams help you visualize the branch structure?
Con il diagramma riesco a capire in modo visuale la struttura della repository evitando
di lavorare su commit privi di un branch perdendo così le modifiche.
